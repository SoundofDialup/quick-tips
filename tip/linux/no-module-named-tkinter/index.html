Error: No module named tkinter

This only appeared when VSCodium had been installed via Flatpack. Remove and install with deb/rpm repo instead (instructions at bottom).

Test in linux terminal

$ python3

>>> import os

>>> import sys

>>> print(os.path.realpath(sys.executable))

Should output something like:

/usr/bin/python3.10

Test in VSCodium terminal

Do the same test as above.

It'll probably output something like:

/usr/bin/python3.11

... because VSCodium is sandboxed!

Install VSCodium via deb/rpm repo

Add the GPG key of the repository:

$ wget -qO - https://gitlab.com/paulcarroty/vscodium-deb-rpm-repo/raw/master/pub.gpg \
| gpg --dearmor \
| sudo dd of=/usr/share/keyrings/vscodium-archive-keyring.gpg

Add the repository:

echo 'deb [ signed-by=/usr/share/keyrings/vscodium-archive-keyring.gpg ] https://download.vscodium.com/debs vscodium main' \
| sudo tee /etc/apt/sources.list.d/vscodium.list

Update then install vscodium:

sudo apt update && sudo apt install codium

Then set the python interpreter in the Command Palette to the one identified in the linux terminal test above.
